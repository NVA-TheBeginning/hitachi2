# ADR-005 - Utilisation d'une API RPC avec oRPC

- **Status** : Accepté
- **Date** : 2026-03-30

## Contexte

Le projet nécessite une communication simple et typée entre frontend et backend.
L'application est interne, donc sans contrainte d'exposition publique de l'API.

## Décision

Nous utilisons une approche RPC avec oRPC pour la communication front-back.

## Alternatives considérées

- API REST classique
- GraphQL

## Conséquences

- Typage de bout en bout
- Réduction du boilerplate
- Développement plus rapide
- Moins standard qu’une API REST
- Couplage plus fort entre frontend et backend
