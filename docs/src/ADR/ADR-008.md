# ADR-008 - Utilisation de pg-boss pour la gestion de la queue

- **Status** : Accepté
- **Date** : 2026-03-30

## Contexte

Le système doit publier des événements (ex: création de réservation) vers une queue afin d’être traités par une autre application (ex: envoi d’e-mail).

## Décision

Nous utilisons pg-boss, basé sur PostgreSQL, pour implémenter la queue.

## Alternatives considérées

- RabbitMQ
- Kafka
- SQS

## Conséquences

- Simplification de l’infrastructure (pas de service externe)
- Intégration directe avec PostgreSQL
- Suffisant pour un volume modéré
- Moins scalable que des solutions dédiées comme Kafka
