# ADR-002 - Architecture monolithe modulaire

- **Status** : Accepté
- **Date** : 2026-03-30

## Contexte

Le projet est développé sur une durée courte avec une équipe restreinte.
Le besoin métier est clair et ne nécessite pas une distribution complexe des services.

## Décision

Nous choisissons une architecture monolithique modulaire.

## Alternatives considérées

- Architecture microservices
- Architecture serverless
- Architecture monolithique

## Conséquences

- Développement plus rapide
- Déploiement simplifié
- Moins de complexité opérationnelle
- Bonne séparation des responsabilités via des modules internes
- Moins scalable qu’une architecture microservices (acceptable pour ce projet)
